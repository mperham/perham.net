Little Guy, Big Pain

Saturday, January 22nd, 2005

I was riding my motorcycle along 2222 about 40mph when suddenly I see something in front of me and BAM it hits my neck HARD! I thought it was a rock or maybe I had been shot cause it hurt like heck. I felt my neck and didn’t see any blood on my glove so I grit my teeth through the pain until I got to the coffee shop. I went to the bathroom, looked at the welt on my neck and discovered a little STINGER in my skin. I had hit a bee and that little bugger did his best to kill me. I’ve never been stung by a bee before and I never want to be again!

One Way to Waste a Saturday Night

Sunday, January 2nd, 2005

My NSLU2 (aka SLUG) is a great little device that provides a file server to my home network without making noise/heat, taking a lot of space, drawing a lot of power, etc. One of the few issues I have with it is that it does not power down the disk. Given that I only use it maybe 5 minutes a day, there’s very little reason why the disk should be spinning the other 23:55. The NSLU2 runs Linux and the way to spin down disks in Linux is through a command called ‘hdparm’. Since the SLUG does not come with hdparm installed, I figured I would need to install the Unslung replacement firmware and then install one of its packages with hdparm in it.

When I initially tried to upgrade the firmware, I did it over wireless and with a HTTP proxy server. Both of these are no-nos when trying to achieve a perfectly reliable HTTP upload with the predictable result: I ‘bricked’ my SLUG. Thankfully there is a recovery mode built into the NSLU2 and by following a set of very arcane instructions I managed to recover from the bungled upgrade. I fixed both network issues and this time the upgrade proceeded smoothly with the end result being that my SLUG is now running Unslung 3.16.

I installed busybox 1.00 and openssh 3.8. Busybox does have hdparm built into it BUT the SLUG mounts the USB drive as a SCSI device and hdparm does not handle power saving with SCSI drives! Argh. This means I will have to pack up my disk and bring it into work so I can plug it into one of the servers there and issue the proper hdparm command via it. At least I have ssh running on the SLUG and so therefore have scp access to the SLUG but what a price to pay for it…
