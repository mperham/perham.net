Standards

Most people agree that Web Services are the new, hot technology that everyone will be using 5 years from now. The problem is, like all technologies, there needs to be standards so that service A can talk to service B. There are tons of standards out there for all different aspects of web services but this can lead to even more problems when each standard has 3 versions published. Vendor A supports SOAP 1.1, UDDI v2 and WSDL 1.0. Vendor B supports SOAP 1.2, UDDI v3 and WSDL 1.0. Even though they support the same standards, they can be incompatible with each other.

This is where the WS-I Basic Profile comes into play. The Basic Profile is an attempt to create a “meta-standard” which codifies a set of standards, the exact versions supported and usage guidelines where specs collide. Why am I telling you all this? Because Webify joined the WS-I and I am Webify’s representative on the Basic Profile working group. The Basic Profile standard is probably THE most important standard for web services interoperability today and I’m very excited to be participating. Read more at http://www.ws-i.org. I will be working on other working groups in the future and attending conferences to further Webify’s goal of being a leader, rather than a follower, in Web Services standards and technologies.
