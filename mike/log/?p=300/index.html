Xdoclet2 - Absolute Garbage

I’ve upgraded all the components in my top secret project to the latest versions so I can try out the latest open source Java stuff. Needless to say it is always good for your long-term job prospects to stay ahead of the curve so you are seen as a leader rather than a follower.

Part of that upgrade is a move to Hibernate3. Since I don’t want to maintain both Java source and Hibernate mapping files, something must allow me to annotate the Java source with the data which needs to be in the mapping file. For the last few years, this something has been Xdoclet1. I have a love-hate relationship with Xdoclet1 - its Hibernate mapping documentation is pretty bad so you often need to spin your wheels searching Google or their mailing list for other examples of what you are trying to do.

Now since Hibernate3 is brand new, I figure I should upgrade to Xdoclet2 also. And thus my misery started. To shorten a long and frustrating story:

  • The project is called Xdoclet 2 v1.0.1. Oh, that seems logical.
  • Their website has NO documentation whatsoever except reference guides to the tags. No examples or samples which tie everything together into something sensical.
  • The product does not work as installed. You have to edit a file deep within it to make it work. They’ve known about this for over a month and not fixed it.
  • Their unit tests (which I dived into as a last ditch attempt to get SOME sort of sample usage) are full of nonsensical variables like “foo” and “bar” with no business scenarios or anything that is easily understandable to anyone but the developer who wrote the test.
  • Their explanation for why I should switch to Xdoclet2? It’s easier for me to write my own plugins! Gee, thanks but 90% of your user base just wants to develop their Hibernate mappings as simple and easily as possible.

Basically I’m giving up and dropping back down to Xdoclet1. As soon as Hibernate Annotations are available, I’m going to drop all this crap and use them instead. I hate to say it but the Xdoclet projects appear to be an example of open source projects that do not think of the user actually trying to use their software. Honestly I don’t see Xdoclet2 around for much longer - those using JDK 1.4 can stick with Xdoclet1 and those with JDK 5 can use the upcoming EJB and Hibernate annotations. Good riddance.

UPDATE: I may think I’m a leader but Hani’s BileBlog got there way before me.
