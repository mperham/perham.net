Microsoft .NET

I’ve been using Visual Studio.NET for a short project I recently started. So far it’s been a mixed experience - the C# and ASP code is fairly similar to Java and JSP so it’s not difficult to understang how things work. One thing I have grown used to over the last few months is a clean and simple IDE that makes writing the code simple and I’m not too impressed with the VS.NET IDE. The best java IDE available is called IDEA and is extremely powerful and easy to use in that it helps a developer write code faster and doesn’t attempt to do things like GUI building (which is really hard to do well). Instead of a clean and simple interface, VS.NET presents the developer with a complex mess of a GUI with 50 buttons, 5 different windows, 100+ menu items and 1000s of help items covering everything from text editing to database design to GUI building but none of which seem to be what you are looking for.

I dream of a day where Microsoft adds five features to a release that are implemented thoroughly and simply instead of fifty features, 45 of which will never be used. You can say the same for any software product: VS.NET, Word, Windows itself.
