A Matter of Time?

Sunday, July 31st, 2005

England has a substantial Muslim community including several “rabid” clerics. So the London tube bombings, while horrible, were not any particular surprise especially given England’s support with the US in Iraq. My question: is France or the Netherlands next? Both also have large Muslim communities with alarming trends in radicalism popping up recently. In France they just expelled a radical cleric accused of holding a terrorist training camp west of Paris and in the Netherlands, a radical Muslim killed the director of a short film which painted the Muslim religion as being misogynist. As the European community will support England in no uncertain terms, it is inevitable that some in those Muslim communities will see this support as making the current “Us vs Them” situation even worse.

All of these communities have the same makeup: they are tight-knit religious communities where many of the members rarely associate outside the community. They do not get to know the country’s people at an individual level but through ignorance and generalization, develop a hatred of the host country’s mores and people. I can’t help but believe that in a few years the current “Terrorists vs the US” will become a more widespread “Terrorists vs the West”. It’s going to get worse before it gets better.

My Latest Foray into Open Source

Thursday, July 28th, 2005

Presenting Light Fixture.

Light Fixture is a Java component which helps you model test scenario data using a simple text descriptor. You describe your beans and Light Fixture will instantiate the beans and optionally persist them to a database at the same time. In essence it allows you to create lightweight test fixtures.

This was something I wrote because I was sick of writing tons of java just to create data for JUnit test scenarios. I hope others will find it useful.

Xdoclet2 - Absolute Garbage

Sunday, July 24th, 2005

I’ve upgraded all the components in my top secret project to the latest versions so I can try out the latest open source Java stuff. Needless to say it is always good for your long-term job prospects to stay ahead of the curve so you are seen as a leader rather than a follower.

Part of that upgrade is a move to Hibernate3. Since I don’t want to maintain both Java source and Hibernate mapping files, something must allow me to annotate the Java source with the data which needs to be in the mapping file. For the last few years, this something has been Xdoclet1. I have a love-hate relationship with Xdoclet1 - its Hibernate mapping documentation is pretty bad so you often need to spin your wheels searching Google or their mailing list for other examples of what you are trying to do.

Now since Hibernate3 is brand new, I figure I should upgrade to Xdoclet2 also. And thus my misery started. To shorten a long and frustrating story:

  • The project is called Xdoclet 2 v1.0.1. Oh, that seems logical.
  • Their website has NO documentation whatsoever except reference guides to the tags. No examples or samples which tie everything together into something sensical.
  • The product does not work as installed. You have to edit a file deep within it to make it work. They’ve known about this for over a month and not fixed it.
  • Their unit tests (which I dived into as a last ditch attempt to get SOME sort of sample usage) are full of nonsensical variables like “foo” and “bar” with no business scenarios or anything that is easily understandable to anyone but the developer who wrote the test.
  • Their explanation for why I should switch to Xdoclet2? It’s easier for me to write my own plugins! Gee, thanks but 90% of your user base just wants to develop their Hibernate mappings as simple and easily as possible.

Basically I’m giving up and dropping back down to Xdoclet1. As soon as Hibernate Annotations are available, I’m going to drop all this crap and use them instead. I hate to say it but the Xdoclet projects appear to be an example of open source projects that do not think of the user actually trying to use their software. Honestly I don’t see Xdoclet2 around for much longer - those using JDK 1.4 can stick with Xdoclet1 and those with JDK 5 can use the upcoming EJB and Hibernate annotations. Good riddance.

UPDATE: I may think I’m a leader but Hani’s BileBlog got there way before me.

Acegi Patch

Saturday, July 23rd, 2005

After testing out Acegi Security the other night, I found a small scalability problem with their anonymous credentials support: accessing a public resource anonymously caused an HttpSession to be created. I submitted a patch to fix the behavior and it was accepted for their upcoming 0.9.0 release:

AnonymousProcessingFilter now has a removeAfterRequest property, which defaults to true. This will cause the anonymous authentication token to be set to null at the end of each request, thus avoiding the expense of creating a HttpSession in HttpSessionContextIntegrationFilter. You may set this property to false if you would like the anoymous authentication token to be preserved, which would be an unusual requirement.
